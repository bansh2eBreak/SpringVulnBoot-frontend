📖 攻击场景

攻击者网站     →   目标 API  
两者 Origin 不同（端口不同），浏览器的 CORS 规则生效。
若目标 API 存在 CORS 配置漏洞，本页面的 JS 可读取受害者的敏感数据。

受害者已登录靶场并持有有效 JWT Token，随后访问了本恶意网站。
本页面 JavaScript 将携带受害者 Token，向靶场 API 发起跨域请求，尝试窃取敏感数据。

📌 三种场景的攻击预期：
① 漏洞1（Origin 完全信任）：addAllowedOriginPattern("*")，任意来源均放行 → 攻击成功
② 漏洞2（Origin 校验可绕过）：addAllowedOriginPattern("*127.0.0.1*")，本意限制本机，
    但本页面 Origin（）也包含 127.0.0.1 → 绕过成功
③ 安全版（严格白名单）：addAllowedOrigin("http://trusted.secnotes.icu")，本页面不在白名单 → CORS 拦截

⚙️ 配置

获取方式：在靶场页面（）按 F12 → 控制台，输入以下命令并复制结果：
localStorage.getItem('Authorization')
